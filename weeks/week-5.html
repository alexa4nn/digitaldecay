Week 5: Metadata for Loss, Failure + Uncertainty

Overview

This week connects research, archives, and interface. We begin by following a rabbit hole, moving through links, images, and references to gather material. This process produces an archive shaped by curiosity, selection, and omission. We then shift to metadata, asking how to describe what is missing, unclear, or unresolved. Finally, we introduce CSS and the Are.na API as tools for structuring and displaying that archive as a webpage.

Activity

Rabbit Hole Archive + Metadata of Absence

Begin with a single entry point, a keyword, image, link, or reference, and follow connections as they emerge. Collect 20-50 items as you move: links, images, screenshots, text fragments, or notes. Organize what you gather into a simple archive using Are.na. Then write metadata that describes not only what is present, but what is missing, unclear, or unresolved in the descriptions. Use the Are.na API to display your archive as a webpage, making the metadata visible alongside the items.

Are.na API Workshop

Displaying your blocks as a webpage

The Are.na API lets you pull your channel's blocks (images, links, text) into a custom HTML page. Public channels require no authentication. The basic workflow:

  1. Find your channel slug (the last part of your Are.na channel URL)
  2. Fetch your blocks from https://api.are.na/v2/channels/YOUR-SLUG
  3. Loop through the blocks and generate HTML for each one
  4. Style the page with CSS to match your archive site

We will use a Google Colab notebook to walk through each step. The notebook fetches your channel data, lets you inspect the block structure, generates a complete HTML page, and downloads it for you to add to your site.

Example: relics, built from this Are.na channel.

CSS Workshop

We will introduce CSS as a way of shaping a website's appearance. Edit your archive webpages with CSS. The goal is not technical complexity, but clarity: how to make your archive visible, and how to represent absence within it.

Assignment: Archive of Digital Loss

The Walk

Conduct an internet walk (minimum 1 hour) following a question-path about digital loss. As you move, collect what you find on an Are.na channel: links, images, screenshots, text fragments, dead URLs, error pages, partial records. Aim for 20-50 items. Save everything, including things that are broken, missing, or only partially accessible.

The Metadata

Go back through your Are.na blocks and write descriptions that function as metadata. For each item, describe not only what it is but what is missing, unclear, or unresolved. Note the source, the date you found it, and what you cannot verify. Use the description field to document absence honestly.

The Webpage

Use the Colab notebook to pull your Are.na channel into a webpage. Style it with CSS to match your archive site. Add the page to your site repository. The metadata you wrote on Are.na should be visible on the page alongside the items.
